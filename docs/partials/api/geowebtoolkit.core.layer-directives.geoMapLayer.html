geoMapLayer
directive in module geowebtoolkit.core

Description

Overview

geoMapLayer adds layer to the page. This tag should be placed within the geoMap tag

Usage

as element:
<geo-map-layer
       layer-attribution="{string|@}"
       layer-name="{string|@}"
       layer-url="{string|@}"
       [layers="{string|@}"]
       [layer-type="{number|@}"]
       wrap-date-line="{string|@}"
       visibility="{string|@}"
       is-base-layer="{string|@}"
       controller-emit-event-name="{string|@}"
       [custom-params="{object|}"]>
</geo-map-layer>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
layerAttributionstring@
  • The id of the element where the map is to be rendered
layerNamestring@
  • A name allocated to the layer for future reference
layerUrlstring@
  • A string value that defines the URL from which the content of the layer will be loaded.
layers
(optional)
string@
  • A comma separated value that tells the remote server which layers of data should is requested form the server. This will be used to make the right request form the remote server that is defined by layerUrl.
layerType
(optional)
number@
  • The type of the data that is being load on the layer. Following types are supported:
    • WMS
    • arcgiscache
    • xyzfilecache
    Note: This directive does not support WFS type. In order to request for WFS type geoFeatureLayer should be used.
wrapDateLinestring@
  • A boolean value ('true', 'false') which defines the map in the layer should be wrapped or not. If wrapped then the map will be unlimited scrollable.
visibilitystring@
  • A boolean value ('true', 'false') for toggling layer on/off
isBaseLayerstring@
  • A boolean value ('true', 'false') telling the server if this layer is base layer or not
controllerEmitEventNamestring@
  • A string value that allocates an AngularJS controller to this layer to be used in JavaScript codes. This will be used in cases that the layer should be controllable with JS codes. Following functions are supported by this controller:
    • hide
    • show
    • setOpacity
customParams
(optional)
object
  • This object is used to support vendor specific query string parameters when requesting layers.
