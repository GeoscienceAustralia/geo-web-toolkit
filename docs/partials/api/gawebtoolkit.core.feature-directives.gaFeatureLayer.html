gaFeatureLayer
directive in module gawebtoolkit.core

Description

Overview

gaFeatureLayer adds layer to the page but only for WFS type of requests. For the other types gaFeatureLayer should be used. This tag should be placed within the gaMap tag.

Dependencies

  • gaMap

Usage

as element:
<ga-feature-layer
       layer-name="{string|@}"
       url="{string|@}"
       post-add-layer="{function|@}"
       controller-emit-event-name="{string|@}"
       on-layer-destroy="{string|@}"
       refresh-layer="{string|@}"
       visibility="{string|@}">
</ga-feature-layer>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
layerNamestring@
  • A name allocated to the layer for future reference
urlstring@
  • A string value that defines the URL from which the content of the layer will be loaded
postAddLayerfunction@
  • Function callback fired after the layer is added
controllerEmitEventNamestring@
  • An string value that will be allocated to this layer as controller. This controller can be called in JS codes in order to control the layer programatically
onLayerDestroystring@
  • Function callback fired on the destruction of a layer
refreshLayerstring@
  • Observed attribute that triggers a layer refresh
  • Following functions are supported by this controller:
    • hide
    • show
    • setOpacity
    • getFeatures
    • addFeature
    • createFeatureAsync
    • createFeature
    • removeFeature
    • isLayerControllerReady
visibilitystring@
  • A boolean value ("true" or "false") that toggles the visibility of the layer on/off
