gaFeatureLayer
directive in module gawebtoolkit.core

Description

Overview

gaFeatureLayer adds layer to the page but only for WFS type of requests. For the other types gaFeatureLayer should be used. This tag should be placed within the gaMap tag.

Dependencies

  • gaMap

Usage

as element:
<ga-feature-layer
       layer-name="{string|@}"
       url="{string|@}"
       [server-type="{number|@}"]
       [wfs-feature-list="{number|@}"]
       wfs-filter-value="{string|@}"
       wfs-version="{string|@}"
       wfs-feature-prefix="{string|@}"
       wfs-feature-type="{string|@}"
       wfs-feature-ns="{string|@}"
       wfs-feature-attributes="{string|@}"
       wfs-geometry-name="{string|@}"
       on-feature-click="{string|@}"
       on-features-loaded="{string|@}"
       update-results-event-name="{string|@}"
       post-add-layer="{string|@}"
       is-lon-lat-order-valid="{string|@}"
       input-format="{string|@}"
       controller-emit-event-name="{string|@}"
       visibility="{string|@}">
</ga-feature-layer>

Directive info

  • This directive creates new scope.
